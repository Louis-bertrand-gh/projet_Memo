Linux & Apache

Commandes Linux (Système)
Fonction Commande Description
Navigation cd /etc/apache2 Se déplacer dans un dossier
Listing ls -lh Lister les fichiers avec détails
Édition sudo nano fichier.conf Modifier un fichier de configuration
Propriété sudo chown -R www-data: /var/www Donner la propriété du dossier à Apache
Droits sudo chmod 755 -R /var/www Appliquer les droits lecture/exécution
Surveillance htop Voir l'utilisation CPU/RAM en temps réel
Espace Disque df -h Vérifier si le disque est plein
Gestion du Service Apache
Action Commande Notes
Démarrer sudo systemctl start apache2 Démarre le service Apache
Arrêter sudo systemctl stop apache2 Arrête complètement Apache
Redémarrer sudo systemctl restart apache2 Redémarrage complet (coupe connexions)
Recharger sudo systemctl reload apache2 Applique config sans coupure
Status sudo systemctl status apache2 Voir l'état du service
Gestion des Sites (VirtualHosts)
Activer un site
sudo a2ensite nom_du_fichier.conf
Désactiver un site
sudo a2dissite nom_du_fichier.conf
Important
sudo systemctl reload apache2
Gestion des Modules
Activer un module
sudo a2enmod nom_module (ex: proxy, ssl, rewrite)
Désactiver un module
sudo a2dismod nom_module
Lister les modules actifs
apache2ctl -M

Reverse Proxy

🧩 1. Préparation - Modules Proxy
Commande Rôle
sudo a2enmod proxy Active le moteur de proxy de base
sudo a2enmod proxy_http Support pour rediriger vers HTTP (Node, Docker, etc.)
sudo a2enmod proxy_balancer Répartition de charge (Load Balancing)
sudo a2enmod proxy_connect Support SSL en tunnel (méthode CONNECT)
📝 2. Directives de Configuration
À placer dans <VirtualHost>
sudo nano /etc/apache2/sites-available/mon-proxy.conf 
ProxyPreserveHost On
ProxyPass / http://127.0.0.1:8080/
ProxyPassReverse / http://127.0.0.1:8080/
🛠️ 3. Tests et Connectivité
Action Commande Utilité
Test de port nc -zv 127.0.0.1 8080 Vérifie si le port écoute
Appel local curl -I http://127.0.0.1:8080 Teste la réponse du service (HTTP 200)
Ports actifs ss -tulpn Liste tous les services écoutants
Logs Apache tail -f /var/log/apache2/error.log Debug 502 Bad Gateway
💡 4. Workflow "Zéro Erreur"
Étape 1 : Modifier
sudo nano /etc/apache2/sites-available/mon-site.conf
Étape 2 : Vérifier syntaxe
sudo apache2ctl configtest
Étape 3 : Appliquer (si OK)
sudo systemctl reload apache2
Étape 4 : Surveiller
sudo tail -n 20 /var/log/apache2/error.log
🔒 Pare-feu (UFW)
Autoriser Apache
sudo ufw allow 'Apache Full'

DNS Autoritaire (BIND9)

🏗️ 1. Installation et Arborescence
Installation
sudo apt install bind9 bind9utils
Chemin Rôle
/etc/bind/named.conf.local Fichier où l'on déclare nos zones (domaines)
/etc/bind/named.conf.options Configuration globale (forwarders, sécurité)
/var/cache/bind/ Répertoire stockage zones répliquées (Esclave)
👑 2. Serveur Primaire (Maître)
Déclaration - named.conf.local
sudo nano /etc/bind/named.conf.local 
zone "mon-domaine.lan" {
    type master;
    file "/etc/bind/db.mon-domaine.lan";
    allow-transfer { IP_DU_SECONDAIRE; };
    also-notify { IP_DU_SECONDAIRE; };
};
Fichier de zone - db.mon-domaine.lan
sudo nano /etc/bind/db.mon-domaine.lan 
$TTL 604800
@   IN  SOA  ns1.mon-domaine.lan. admin.mon-domaine.lan. (
                  2023102701 ; Serial (À incrémenter)
                  604800     ; Refresh
                  86400      ; Retry
                  2419200    ; Expire
                  604800 )   ; Negative Cache TTL

@   IN  NS   ns1.mon-domaine.lan.
@   IN  NS   ns2.mon-domaine.lan.
ns1 IN  A    IP_SERVEUR_MAITRE
ns2 IN  A    IP_SERVEUR_ESCLAVE
www IN  A    IP_DU_SERVEUR_WEB
👥 3. Serveur Secondaire (Esclave)
Déclaration esclave - named.conf.local
sudo nano /etc/bind/named.conf.local 
zone "mon-domaine.lan" {
    type slave;
    file "/var/cache/bind/db.mon-domaine.lan";
    masters { IP_DU_MAITRE; };
};
🛠️ 4. Commandes de Vérification
Action Commande Utilité
Syntaxe globale sudo named-checkconf Vérifie tous les fichiers Config
Fichier de zone sudo named-checkzone mon-domaine.lan /etc/bind/db.mon-domaine.lan Valide le fichier de zone
Recharger zones sudo rndc reload Applique changements BIND
Logs transfert tail -f /var/log/syslog Debug transfert zone (Master)
🔍 5. Tests et Diagnostic
Tester le serveur local
dig @localhost www.mon-domaine.lan
Test transfert (AXFR)
dig axfr mon-domaine.lan @IP_DU_MAITRE
Vérifier SOA
dig mon-domaine.lan SOA
📂 Chemins Importants
Aller aux configurations BIND
cd /etc/bind/
Aller aux zones répliquées
cd /var/cache/bind/
Aller aux logs
cd /var/log/

Commandes de Test

🐧 1. Commandes Réseau & DNS (Linux / Universel)
Commande Utilité
ping [IP/Domaine] Vérifie la connectivité ICMP de base
dig [domaine] Interroge le serveur DNS par défaut pour un domaine
dig @[IP_serveur] [domaine] Teste un serveur DNS spécifique (ex: @8.8.8.8)
nslookup [domaine] Alternative simple à dig pour tester la résolution de nom
host -t A [domaine] Trouve l'adresse IPv4 associée à un nom
traceroute [domaine] Affiche chaque saut (routeur) entre vous et la destination
nc -zv [IP] [Port] Netcat : vérifie si un port est ouvert (ex: nc -zv 1.1.1.1 443)
curl -I [URL] Récupère uniquement l'en-tête HTTP (codes 200, 404, 502)
🌐 2. Apache & Nginx (Serveurs Web / Reverse Proxy)
⚠️ Avant de redémarrer, toujours tester la syntaxe !
Apache
Commande Utilité
apache2ctl configtest Vérifie la syntaxe de tous les fichiers .conf
apache2ctl -S Affiche la hiérarchie des VirtualHosts actifs
apache2ctl -M Liste tous les modules chargés (proxy, ssl, rewrite)
tail -f /var/log/apache2/error.log Analyse les erreurs en temps réel
Nginx
Commande Utilité
nginx -t Teste la syntaxe des fichiers de configuration
nginx -T Affiche la configuration complète consolidée
nginx -V Affiche la version et les arguments de compilation
tail -f /var/log/nginx/error.log Indispensable pour diagnostiquer les « 502 Bad Gateway »
🛠️ 3. BIND9 (DNS Autoritaire & Secondaire)
⚠️ Le DNS est sensible : une virgule manquante et le service ne démarre plus
Commande Utilité
named-checkconf Vérifie les fichiers de configuration globaux
named-checkzone [domaine] [chemin] Valide le fichier de zone (Serial, enregistrements A, MX)
rndc status Affiche l'état de santé du démon BIND
rndc reload Recharge les zones sans redémarrer
dig axfr @[Master_IP] [domaine] Teste le transfert de zone (Primaire → Secondaire)
🪟 4. Windows (PowerShell & CMD)
Commande Utilité
ipconfig /all Affiche toute la configuration réseau (IP, DNS, Passerelle)
ipconfig /flushdns Vide le cache DNS local (après changement de zone)
nslookup -type=soa [domaine] Vérifie l'autorité DNS pour un domaine
Test-NetConnection -ComputerName [IP] -Port [Port] PowerShell : équivalent moderne de Telnet/Netcat
tracert [IP] L'équivalent Windows de traceroute
Get-Service [Nom] PowerShell : vérifie l'état d'un service (ex: Get-Service DNS)
📊 5. Commandes Système Linux (Performance & Ports)
Commande Utilité
ss -tulpn Liste tous les ports ouverts et les programmes qui les utilisent
journalctl -u [service] -f Affiche les logs système d'un service (ex: -u bind9)
top ou htop Surveille la consommation de ressources
df -h Vérifie qu'une partition n'est pas pleine
lsof -i :80 Trouve quel processus occupe le port 80
